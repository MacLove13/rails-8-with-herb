# General
- Avoid complex code; always opt for simpler, more functional code with low complexity.

# Database
- The database should always use UUIDs for Primary Keys.

# CI / Tests
- Test coverage should be 100%.
- You must test and ensure that all tests are working and that there are no errors or warnings.

# Fron-End changes
- Any changes made to the Front End must include screenshots in the Pull Request demonstrating the before and after.
